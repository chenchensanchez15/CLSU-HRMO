Google Drive Integration Setup Guide
====================================

To implement Google Drive file storage in the HRMO system, follow these steps:

1. Enable GD Extension in PHP:
   - Open your php.ini file (usually in C:\xampp\php\php.ini)
   - Find the line ";extension=gd" and remove the semicolon to uncomment it
   - Change it to: extension=gd
   - Restart your Apache server in XAMPP Control Panel

2. Install Google API Client Library:
   - Run the following command in your project directory:
     composer require google/apiclient

3. Set up Google Cloud Project:
   - Go to https://console.cloud.google.com/
   - Create a new project or select an existing one
   - Enable the Google Drive API
   - Go to Credentials section
   - Create a Service Account
   - Download the JSON key file
   - Rename it to "google_credentials.json"
   - Place it in: c:\xampp\htdocs\HRMO\writable\credentials\

4. Configure Environment Variables:
   - Add these to your .env file:
     GOOGLE_CREDENTIALS_PATH = writable/credentials/google_credentials.json
     GOOGLE_DRIVE_FOLDER_ID = your_folder_id_here (optional)

5. Update the Database:
   - Modify the applicant_documents table to store Google Drive file IDs instead of local file paths
   - Change the filename columns to store the Google Drive file IDs

6. Update Controllers:
   - Modify the Account::updateFile method to use GoogleDriveService
   - Update the File::viewFile method to retrieve from Google Drive
   - Update delete functionality to remove from Google Drive

7. Security Considerations:
   - Ensure proper access controls are in place
   - Consider using short-lived access tokens instead of 'anyone' permissions
   - Implement proper error handling for API failures

8. Testing:
   - Test file uploads to ensure they go to Google Drive
   - Test file viewing to ensure they can be retrieved
   - Test file deletion to ensure they are removed from Google Drive
   - Verify that all existing functionality continues to work

Note: This is a significant change that will affect the file storage and retrieval mechanisms throughout the application. Make sure to backup your database and files before implementing these changes.
